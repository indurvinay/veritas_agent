[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's look at the architecture. When a user sends a query, it starts at the Security Checkpoint node. This node scrubs personal data like emails and SSNs and blocks any prompt injections or credential requests. If it passes, the query goes to the Orchestrator Router. The Orchestrator decides whether to delegate the work to the Email Triage Agent, the Calendar Scheduler Agent, or the Workspace Search Agent. 

All three agents communicate with our simulated workspace environment through a Model Context Protocol server, running as a local stdio process. Before executing any mutations—like sending a drafted reply or booking a meeting—the workflow routes through a Human-in-the-loop Review Node, which pauses the graph and asks for explicit user approval. Only after confirmation does it flow to the Final Output Node.
